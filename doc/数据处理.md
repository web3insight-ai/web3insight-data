# Web3Insights 数据处理说明文档

本说明文档覆盖 Web3Insights 项目中 GitHub 数据处理的完整流程，共分五个步骤



## 步骤 1：下载数据（gharchive）

**文件** : `1.gharchive_downloader.sh`

**功能** : 
- 从 https://data.gharchive.org 下载指定日期范围的 JSON.GZ 格式 GitHub 数据
- 自动按 `Data/raw/YYYY/MM/DD/` 结构保存
- 支持：
  - 下载单天 `--from YYYY-MM-DD --to YYYY-MM-DD`
  - 下载全年（开发中）
  - 日志输出至 `Data/logs/gharchive_download.log`

**示例** : 
```bash
./1.gharchive_downloader.sh --from 2025-09-01 --to 2025-09-03
```

---

## 步骤 2：解压数据

**文件** : `2.decompress.sh`

**功能** : 
- 解压 `.json.gz` 为 `.json`，并保留原始目录结构
- 支持多线程解压（默认 8 并发）
- 日志记录至 `Data/logs/decompress.log`
- 若 `.json` 解压失败，会自动删除错误文件

**示例** : 
```bash
./2.decompress.sh -d 2025/09/01        # 解压某天
./2.decompress.sh -m 2025/09           # 解压某月
./2.decompress.sh -y 2025              # 解压某年
```

---

## 步骤 3：清洗数据

**文件** : `3.clean.py`

**功能** : 
- 读取解压后的 `.json` 文件
- 移除无效字段、空对象、结构异常的 JSON 数据
- 清洗结果写入 `Data/cleaned/YYYY/MM/DD/*.cleaned.json`
- 多进程并发，日志记录至 `Data/logs/clean.log`

**示例** : 
```bash
python3 3.clean.py -d 2025/09/01
python3 3.clean.py -m 2025/09
```

---

## 步骤 4：提取结构化数据

**文件** : `4.extract.py`

**功能** : 
- 将 `.cleaned.json` 提取为三类结构化表：
  - `actors`：用户信息（Actor 节点）
  - `repos`：仓库信息（Repo 节点）
  - `events`：交互关系（INTERACTS_WITH 边）
- 输出路径：
  - `Data/structured/actors/YYYY/MM/DD.csv`
  - `Data/structured/repos/YYYY/MM/DD.csv`
  - `Data/structured/events/YYYY/MM/DD.csv`
- 支持并行处理、日志记录 `Data/logs/extract.log`

**示例** : 
```bash
python3 4.extract.py -f Data/cleaned/2025/09/01/2025-09-01-1.cleaned.json
python3 4.extract.py -d 2025/09/01
```

---

## 步骤 5：导入 PostgreSQL

**文件** : `5.import_pgsql.sh`

**功能** : 
- 将 `structured/actors/`, `structured/repos/`, `structured/events/` 中的 CSV 导入 PostgreSQL 数据库
- 目标表结构：
  - `actors(id, login, type, site_admin)`
  - `repos(id, name, full_name, language)`
  - `events(actor_id, repo_id, type, created_at)`
- 支持导入指定年月日的数据
- 使用 `psql` 命令行工具执行 COPY 导入

**示例** : 
```bash
./5.import_pgsql.sh -d 2025/09/01
```

---

## 📁 项目目录结构（简略）
```bash
Data/
├── raw/            # 原始下载 .json.gz
├── extracted/      # 解压后的 .json
├── cleaned/        # 清洗后的 .cleaned.json
├── structured/     # 提取的 CSV 结构化数据（actors/repos/events）
├── logs/           # 每步处理日志
```

---

## 🧪 推荐运行顺序

```bash
./1.gharchive_downloader.sh --from 2025-09-01 --to 2025-09-01
./2.decompress.sh -d 2025/09/01
python3 3.clean.py -d 2025/09/01
python3 4.extract.py -d 2025/09/01
./5.import_pgsql.sh -d 2025/09/01
```
